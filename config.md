## Root configuration object

| Name | Type | Description | |
| --- | --- | --- | --- |
| <a id="config-opt-tsnet-hostname"></a>`tsnet.hostname` | string | Hostname to use for the tsnet node.| Default: _"tsiam"_ |
| <a id="config-opt-tsnet-authkey"></a>`tsnet.authKey` | string | AuthKey can be used to authenticate the tsnet node automatically.<br>If empty, tsnet will rely on existing state in the database.|  |
| <a id="config-opt-tsnet-statedir"></a>`tsnet.stateDir` | string | Directory where tsnet stores its state.<br>If empty, defaults to a folder next to the loaded config file.|  |
| <a id="config-opt-tsnet-ephemeral"></a>`tsnet.ephemeral` | boolean | If true, the tsnet node is ephemeral (not persisted in the tailnet).| Default: _false_ |
| <a id="config-opt-tsnet-funnel"></a>`tsnet.funnel` | boolean | If true, enables Tailscale Funnel to expose the .well-known endpoints publicly.<br>This allows external OIDC clients to discover the JWKS without being on the tailnet.<br>Note: see requirements for enabling Tailscale Funnel: https://tailscale.com/kb/1223/funnel| Default: _false_ |
| <a id="config-opt-logs-level"></a>`logs.level` | string | Controls log level and verbosity. Supported values: `debug`, `info` (default), `warn`, `error`.| Default: _"info"_ |
| <a id="config-opt-logs-omithealthchecks"></a>`logs.omitHealthChecks` | boolean | If true, calls to the healthcheck endpoint (`/healthz`) are not included in the logs.| Default: _true_ |
| <a id="config-opt-logs-json"></a>`logs.json` | boolean | If true, emits logs formatted as JSON, otherwise uses a text-based structured log format.<br>Defaults to false if a TTY is attached (e.g. when running the binary directly in the terminal or in development); true otherwise.|  |
| <a id="config-opt-tokens-lifetime"></a>`tokens.lifetime` | duration | Token life time, as a Go duration<br>Must be between 1 and 60 minutes| Default: _"5m"_ |
| <a id="config-opt-signingkey-storage"></a>`signingKey.storage` | string | Key storage provider<br>Allowed values: "file" (default), "memory", "AzureKeyVaultKeys", "AzureKeyVaultSecrets"| Default: _"file"_ |
| <a id="config-opt-signingkey-algorithm"></a>`signingKey.algorithm` | string | Signing algorithm to use. Supported values: RS256, ES256, ES384, ES512, EdDSA.| Default: _"ES256"_ |
| <a id="config-opt-signingkey-curve"></a>`signingKey.curve` | string | Curve for EdDSA algorithm.<br>Currently only Ed25519 is supported.|  |
| <a id="config-opt-signingkey-file-storagepath"></a>`signingKey.file.storagePath` | string | StoragePath is the path to store signing key on disk.<br>The key is not encrypted on disk.| **Required** |
| <a id="config-opt-signingkey-azurekeyvaultkeys-vaulturl"></a>`signingKey.azureKeyVaultKeys.vaultUrl` | string | VaultURL is the URL of the Azure Key Vault (e.g., https://myvault.vault.azure.net/).| **Required** |
| <a id="config-opt-signingkey-azurekeyvaultkeys-keyname"></a>`signingKey.azureKeyVaultKeys.keyName` | string | KeyName is the name of the key in Azure Key Vault used for wrapping/unwrapping.<br>The key is unwrapped using Azure Key Vault on app startup.| **Required** |
| <a id="config-opt-signingkey-azurekeyvaultkeys-storagepath"></a>`signingKey.azureKeyVaultKeys.storagePath` | string | StoragePath is the path to store the wrapped signing key on disk.| **Required** |
| <a id="config-opt-signingkey-azurekeyvaultkeys-tenantid"></a>`signingKey.azureKeyVaultKeys.tenantId` | string | TenantID is the Azure AD tenant ID for authentication.<br>If empty, DefaultAzureCredential will be used (which can leverage authentication methods including: environmental variables, managed identity, workload identity).|  |
| <a id="config-opt-signingkey-azurekeyvaultkeys-clientid"></a>`signingKey.azureKeyVaultKeys.clientId` | string | ClientID is the Azure AD application (client) ID for authentication.<br>If empty, DefaultAzureCredential will be used (which can leverage authentication methods including: environmental variables, managed identity, workload identity).|  |
| <a id="config-opt-signingkey-azurekeyvaultkeys-clientsecret"></a>`signingKey.azureKeyVaultKeys.clientSecret` | string | ClientSecret is the Azure AD application client secret for authentication.<br>If empty, DefaultAzureCredential will be used (which can leverage authentication methods including: environmental variables, managed identity, workload identity).|  |
| <a id="config-opt-signingkey-azurekeyvaultsecrets-vaulturl"></a>`signingKey.azureKeyVaultSecrets.vaultUrl` | string | VaultURL is the URL of the Azure Key Vault (e.g., https://myvault.vault.azure.net/).| **Required** |
| <a id="config-opt-signingkey-azurekeyvaultsecrets-secretname"></a>`signingKey.azureKeyVaultSecrets.secretName` | string | SecretName is the name of the secret in Azure Key Vault that stores the signing key.<br>The entire key is stored in Azure Key Vault.| **Required** |
| <a id="config-opt-signingkey-azurekeyvaultsecrets-tenantid"></a>`signingKey.azureKeyVaultSecrets.tenantId` | string | TenantID is the Azure AD tenant ID for authentication.<br>If empty, DefaultAzureCredential will be used (which can leverage authentication methods including: environmental variables, managed identity, workload identity).|  |
| <a id="config-opt-signingkey-azurekeyvaultsecrets-clientid"></a>`signingKey.azureKeyVaultSecrets.clientId` | string | ClientID is the Azure AD application (client) ID for authentication.<br>If empty, DefaultAzureCredential will be used (which can leverage authentication methods including: environmental variables, managed identity, workload identity).|  |
| <a id="config-opt-signingkey-azurekeyvaultsecrets-clientsecret"></a>`signingKey.azureKeyVaultSecrets.clientSecret` | string | ClientSecret is the Azure AD application client secret for authentication.<br>If empty, DefaultAzureCredential will be used (which can leverage authentication methods including: environmental variables, managed identity, workload identity).|  |
